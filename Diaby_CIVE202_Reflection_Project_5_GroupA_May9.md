**Project \#5 Reflection**

In this project, we completed a full analysis from extracting data, to writing outlines, and to constructing our whole code on our own, there were many things that we learned.

We learned to create a table based on a dataframe/dataset to include only the columns we need. This is useful to show specific data from the dataset rather than displaying the whole thing and having to talk about what information the viewer should be looking at. In our code, we used an already filtered dataset and made a table using only the columns we needed. Another thing we learned was converting dataframes into Python sets. We wanted to figure out a way to find the differences in the two dataframes we were comparing. Python sets allowed us to find what countries, under a certain service level, were present in one year but not in another. We also learned how to make our graphs interactive. You can only plot so much data on graphs, so interactive graphs helped us complete the story we were trying to tell. On one of our graphs, we had our hover data as population, the y-axis as coverage, and the x-axis as the countries in a specific year. Not only can the viewer see how coverage for a water source condition changes between countries and residence type, but they can also see the trends in population.

I think our Project \#5 topic really had us think about how data should be displayed and what kinds of information the client would be interested in. There were some things in the code that we included even though the client never explicitly asked for them. Out intention is not to bombard them, but to introduce them to other things they should consider and other relevant information they hadn’t thought about that were actually really useful in our analysis. We wanted to give the client as many perspectives as possible while not straying away from the task at hand.
